## 二、我的公众号工作流，完整拆解

在用工具之前，最重要的还是想清楚定位——你的公众号给谁看、写什么、凭什么让人关注。工具解决的是效率问题，不是方向问题。

还有不同渠道喜欢的风格也是不同的，在公众号这篇，详细的教程类的更受欢迎，能够触达大家的转发点；小红书那边一定要非常漂亮审美好，情绪渲染＞信息密度。

下面详细说说我围绕公众号的实际工作流，每一步都是真实操作。

![[笔记同步助手/images/4a9a70194a686c83f3ae422bd79d5034_MD5.png]]

  

### 第一步：素材收集——两个非常重要小技巧

好文章不是坐下来硬想出来的，是做事过程中自然攒出来的。我有两个比较有效的素材收集习惯，我认为这个是最终出稿质量的决定性因素之一。

**技巧一：知识管理。**

我会把一个项目的所有相关文件汇总到一个文件夹里。每次写文章，把这个文件夹的路径给 Claude Code，它就有了完整的上下文。

比如我写 pdf2app 相关的文章，会先打开 pdf2app 的产品文件夹，让 Claude Code 读完所有背景材料再动笔。

**这不是什么高深的方法，就是"把材料整理好再开工"。** 但很多人用 AI 写东西的时候，直接甩一句话让它写，给的上下文太少，写出来自然泛泛而谈。

**技巧二：对话记录就是素材。**

我们每次和 Claude Code 对话都有完整的对话记录。这些记录本身就是最好的创作素材——因为里面有完整的思考过程、踩坑经历、灵光一现的时刻。

我的操作是这样的：做完一个项目之后，我会跟 Claude 说：

​

> “这个东西的创造过程，我想写一篇文章。请你分析我做这个项目时和你所有相关的对话文档，帮我把细节写下来，输出一个 md 文档，详细展示所有过程。”

如果我要写别人做的项目，也是同样的思路——请对方把上面那段话发给自己的 Claude Code，导出一份完整的过程记录发给我。这样我的 Claude Code 就能基于一手素材写出所有细节。

Claude 会翻之前的对话记录，把关键节点、决策过程、踩坑细节都梳理出来。**这些一手的过程记录，比事后回忆写出来的东西真实得多。**

![[笔记同步助手/images/e0b6ccd1cd889e10b8c6ac6a12715403_MD5.png]]

  

### 第二步：让 Claude 出稿

素材准备好了，Skill 也装了，接下来就是让 Claude 开始写。

说"出稿"，或者把具体需求告诉它，Claude 会按 Skill 里定义的写作框架生成文章。它会自动判断内容类型——教程类用六段式框架（先看结果→概念→操作→实战→拿走即用→写在最后），深度长文用四幕式框架（序言→铺设→论述→收束）——然后套用对应的写作规范。

**但是——**

​

### 第三步：你是监工，这一步最重要

**Claude 生成的初稿，你绝对不能直接发。**

这不是因为 Claude 写得差，而是因为它有几个改不掉的毛病：

![[笔记同步助手/images/85af8c219875f5326237fe61f5d97415_MD5.png]]

**你的角色是流水线上的监工。** Claude 是那个产能极高但审美有偏差的工人——它能快速搭建文章框架、整理信息、填充细节，但最终的质量把控必须是你来做。

具体来说，我审稿时重点看三件事：

1.  1\. **标题** — 准不准确？有没有夸大？能不能 10 秒内让读者判断值不值得看？
    
2.  2\. **文章结构** — 逻辑通不通？信息密度够不够？有没有废话段落？
    
3.  3\. **阐述的清晰度** — 每个概念是否用人话解释了？读者能不能看懂并照着做？
    

审完之后让 Claude 改，把你的要求直接写下来，改完再看，通常 2-3 轮就能到发布标准。

这个其实最最重要的，你要非常细颗粒度地给出反馈和修改建议。

​

### 小技巧：写文章请用 Opus 4.6

这条经验是真金白银买来的。

有一次我写一篇文章，用 Sonnet 来回改了 3 个小时，就是写不出我满意的东西——结构松散、重点不突出、说了一堆正确的废话。

换成 Opus 4.6，30 分钟搞定。结构清晰、重点明确、不废话。

写文章是需要"理解力"的任务——理解你的意图、理解读者的需求、理解信息的主次。Sonnet 在这件事上和 Opus 有明显差距。

我的用法：日常跑脚本、调配置用 Sonnet（便宜、够用），写文章、做内容一律切 Opus。

![[笔记同步助手/images/2e3e1baac5dc9a9d624bfdea713fb7d3_MD5.png]]

  

### 第四步：排版 + 头图 + 配图

文章定稿之后，说"排版"，Claude 会调用排版工具把 Markdown 转成公众号 HTML（01fish 主题：墨绿+鱼红+宣纸底）。再说"做头图"、“做配图”，生成对应的 HTML 文件，浏览器打开下载 PNG。

这一步基本是全自动的，偶尔需要微调一下头图上的文案。

​

### 第五步：发布

Skill 里有 `/distribute` 一键分发的功能，能通过 API 直推公众号草稿箱。配置方法很简单——跟 Claude Code 说"我想把文章直接推送到公众号草稿箱，请引导我配置"，它会一步一步教你操作。

这一步skill的控制力有时候会不够，只同步了内容但没有同步配图的情况出现很多。这时候，你就再加一个指令，请把头图，配图以及内容，一并同步到草稿箱当中。

我这个skill的配图就是我当前这个公众号所用的风格。
